CAMBIOS REALIZADOS:

- Se ha eliminado el modificador "readonly" de las variables
- No se permiten expresiones de más de dos operadores en las que uno de ellos sea la llamada a una función
- El valor devuelto por la llamada a una función se puede asignar a una variable
- No existen los contenedores de ristras, y tampoco la operación '+' de las mismas
- La importación de librerías ahora separa cada carpeta de la ruta con /
- En Flex cada tipo de dato numérico tiene un TOKEN exclusivo para él
- No existe el casteo, por lo que el analizador produce un error si encuentra tipos no coincidentes

ADVERTENCIAS:

 - Los mensajes de error no están diseñados para el usuario final, sino para los desarrolladores del lenguaje
 - La importación de librerías aún no está implementada, pero se tiene pensado incluirlo a futuro
